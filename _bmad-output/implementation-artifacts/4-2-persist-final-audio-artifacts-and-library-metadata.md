# Story 4.2: Persist Final Audio Artifacts and Library Metadata

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a user who completed conversion,
I want final audio files and metadata to be saved in my local library,
so that I can reliably find and replay generated audiobooks later.

## Acceptance Criteria

1. **Given** a final MP3 or WAV output is generated by `audio_postprocess_service.py`  
   **When** persistence runs through `library_service.py`  
   **Then** the audio file path is stored under local runtime library conventions  
   **And** library metadata is written to SQLite as source of truth.

2. **Given** metadata persistence uses `library_items_repository.py`  
   **When** a new audiobook record is created  
   **Then** required fields include title, source, format, engine, voice, language, duration, and created timestamp  
   **And** all persisted fields follow `snake_case` naming and ISO-8601 UTC time format.

3. **Given** persistence can fail due to disk or database errors  
   **When** write operations fail  
   **Then** the service returns normalized `{ok, data, error}` with structured error object  
   **And** partial writes are avoided using transactional behavior.

4. **Given** observability is required for library integrity  
   **When** library item creation succeeds or fails  
   **Then** JSONL events are emitted via `jsonl_logger.py` with `stage=library_persistence`  
   **And** events include `correlation_id`, `job_id`, `event`, `severity`, and UTC ISO-8601 `timestamp`.

## Dev Notes

### Developer Context Section

- Story 4.2 depends directly on output produced by [AudioPostprocessService.assemble_and_render()](src/domain/services/audio_postprocess_service.py:44) and launched through [TtsOrchestrationService.launch_conversion()](src/domain/services/tts_orchestration_service.py:113).
- Current codebase already persists conversion/chunk lifecycle in SQLite repositories and currently contains a stub [LibraryItemsRepository](src/adapters/persistence/sqlite/repositories/library_items_repository.py:10); this story must turn that boundary into working persistence for library records.
- The conversion flow currently computes final target artifact under `runtime/library/audio/{job_id}.{format}` in [TtsOrchestrationService.launch_conversion()](src/domain/services/tts_orchestration_service.py:137). Story 4.2 must persist that resulting artifact metadata immediately after post-processing success.
- Do not move fallback policy or chunk orchestration out of [TtsOrchestrationService](src/domain/services/tts_orchestration_service.py:77). Library persistence is a downstream responsibility.
- Keep all response contracts normalized via [success()](src/contracts/result.py:86), [failure()](src/contracts/result.py:94), and structured error envelopes expected across existing services.

### Technical Requirements

- Implement a dedicated [LibraryService](src/domain/services/library_service.py) domain service that persists one library item per successful final artifact and returns normalized [Result](src/contracts/result.py:11).
- Persist metadata in SQLite through [LibraryItemsRepository](src/adapters/persistence/sqlite/repositories/library_items_repository.py:10) with explicit required fields:
  - `job_id`
  - `title`
  - `source_path`
  - `audio_path`
  - `format`
  - `engine`
  - `voice`
  - `language`
  - `duration_seconds`
  - `byte_size`
  - `created_at`
- Enforce `snake_case` payload keys and ISO-8601 UTC timestamps at the service boundary before persistence.
- Guarantee transactional write semantics in repository insertion path (single `BEGIN`/`COMMIT`, rollback on failure).
- Treat persistence failure as conversion completion failure for this story scope (do not silently drop metadata while reporting success).
- Emit observability events at stage `library_persistence` with stable names like:
  - `library.item_created`
  - `library.item_create_failed`
- Carry required event fields: `correlation_id`, `job_id`, `event`, `stage`, `severity`, `timestamp`.
